Quando um processo for dividido em etapas e envolver ChatGPT, Codex, Antigravity, GitHub, arquivos locais, navegador, chats limpos, regras/memórias, backups, extensão Chat Fácil ou outra ferramenta externa, criar primeiro um pacote operacional revisável em ZIP. O pacote deve conter roteiro, estrutura JSON, prompts separados por destino, checklist de validação, critérios de parada e instruções de segurança.

O ChatGPT atual deve planejar, resumir, criar os prompts, organizar o ZIP e mostrar uma prévia para o usuário aprovar. O Codex deve atuar como orquestrador/integrador quando tiver acesso completo: ler o roteiro aprovado, localizar arquivos, usar uma conversa limpa do ChatGPT autorizada como bancada operacional, colar prompts nos destinos corretos, acionar Antigravity quando necessário, salvar backups no GitHub, atualizar índices e devolver relatórios. O Codex deve evitar gastar créditos com programação desnecessária; sua prioridade é executar, copiar, colar, organizar, versionar e integrar.

A conversa limpa do ChatGPT deve ser considerada autorizada quando tiver sido criada ou indicada para o processo e estiver logada na conta do usuário. O Codex não precisa pedir autorização a cada uso dessa conversa limpa se ela fizer parte do roteiro aprovado. Deve pedir autorização apenas se precisar usar conversas antigas, chats ativos do usuário, áreas sensíveis, ações fora do roteiro, apagamentos, substituições ou acessos que possam interferir no trabalho atual do usuário.

O Antigravity deve ser priorizado para programação, mockups, testes no workspace e evolução de extensões/aplicações. Todo roteiro deve separar claramente: o que é para Codex, o que é para Antigravity, o que é para ChatGPT limpo, o que é para GitHub e o que exige ação do usuário. O objetivo é reduzir trabalho manual, evitar perda de contexto, economizar créditos/tokens do Codex, preservar as conversas ativas do usuário e manter backups/versionamento no GitHub sem sobrescrever versões antigas.
