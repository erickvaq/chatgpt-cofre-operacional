## Seguranca

Nao envie ao GitHub senhas, tokens, cookies, dados sensiveis, prints privados ou arquivos pessoais fora do escopo. Nao apague, substitua, sobrescreva, restaure, publique, mova arquivos importantes, altere memorias, exponha dados ou troque versoes funcionais sem confirmacao explicita.

Comandos curtos como ok, continue, proximo, próximo, avance e execute autorizam apenas o proximo passo seguro planejado. Eles nao autorizam acao destrutiva, publicacao externa, exposicao de dados ou substituicao de versao funcional.

## Organizacao

Quando uma conversa ou processo gerar arquivos, use uma pasta propria. Se GitHub nao estiver autorizado, use `C:\Users\Windows User\Documents\Codex\CONVERSAS_ORGANIZADAS\`. Se GitHub estiver autorizado, use `erickvaq/chatgpt-cofre-operacional/codex_conversas/` como espelho, backup e versionamento.

Sempre crie e valide a camada local primeiro. Quando houver autorizacao e acesso ao GitHub, prepare ou execute a camada GitHub como espelho, backup, historico, fonte alternativa e local de links permanentes.

Publique ou sincronize GitHub somente se todas as condicoes forem verdadeiras: autorizacao explicita para GitHub; repositorio correto identificado; arquivos locais criados e validados; backups existentes; destino GitHub claro; conteudo sem senhas, tokens, cookies, dados sensiveis, prints privados ou arquivos fora do escopo; ausencia de sobrescrita perigosa sem comparacao; relatorio final com itens enviados, destino e links.

Mantenha `links\LINKS_GITHUB_E_LOCAL.md` quando houver material sincronizavel. Esse arquivo deve listar caminho local, caminho GitHub planejado, link GitHub real somente se publicado, status, observacoes de seguranca e proxima acao segura. Nao invente link de arquivo nao publicado.
