# REGRA — Salvar regra na memória persistente e no GitHub

## Chave canônica
`salvar_regra_memoria_persistente_e_github`

## Gatilhos naturais
Quando o usuário disser, com esse sentido, expressões como:

- `salve como regra`
- `salvar regra`
- `salve isso`
- `lembre disso`
- `lembre o que eu disse`
- `guarde isso`
- `isso é regra`
- `atualize a memória`
- `atualizar memória`
- `registre isso`
- `deixe isso salvo`
- `salve canonicamente`
- `salve como regra acessível por gatilhos`

## Regra operacional
Interpretar como pedido real de salvamento/consolidação de regra. O salvamento deve priorizar a memória persistente do ChatGPT quando disponível, e o GitHub deve funcionar como cofre, backup, histórico e ponte operacional — não como substituto único da memória persistente.

## Ordem de execução
1. Reconstruir a intenção do usuário e transformar em regra curta, canônica, acionável e acessível por gatilhos naturais.
2. Verificar se já existe regra equivalente ou relacionada.
3. Consolidar/atualizar regra existente quando couber, evitando duplicidade.
4. Tentar salvar/atualizar na memória persistente do ChatGPT quando o ambiente permitir.
5. Salvar também no GitHub/cofre operacional quando a regra for importante, reutilizável ou fizer parte do sistema de trabalho.
6. Informar status separado:
   - memória persistente confirmada, somente se houver cartão `Memória atualizada`, botão `Gerenciar` ou conferência manual;
   - memória persistente não confirmada, quando não houver prova visual;
   - GitHub salvo, quando houver commit/link do cofre;
   - regra ativa nesta conversa, quando só estiver valendo no contexto atual.
7. Entregar a regra final em formato limpo e copiável.

## O que evitar
- Não salvar somente no GitHub quando o usuário pediu memória persistente.
- Não afirmar `memória persistente confirmada` sem evidência visual ou conferência manual.
- Não criar duplicata se a melhoria couber em regra já existente.
- Não substituir regras antigas sem escopo claro e sem preservar o que continua válido.

## Critério de sucesso
A regra fica clara, curta, acionável por gatilhos naturais, preserva regras atuais, tenta usar memória persistente quando disponível, registra backup no GitHub quando útil e informa o status real sem fingir confirmação.
