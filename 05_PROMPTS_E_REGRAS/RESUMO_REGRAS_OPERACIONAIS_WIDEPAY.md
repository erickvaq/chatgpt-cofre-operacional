# RESUMO DAS REGRAS OPERACIONAIS WIDEPAY

Este documento apresenta um resumo compacto das regras mais importantes do projeto para consulta rápida.

1. **WidePay é Fonte Principal:** Toda situação financeira atual de carnês e cobranças deve ser consultada em tempo real no WidePay.
2. **Contratos Locais são Apenas Apoio:** A pasta `C:\Users\Windows User\Desktop\AGUA VIVA` deve ser usada apenas para complementar dados contratuais (nome, lote, quadra, quantidade de parcelas). O WidePay prevalece.
3. **Não Acessar Configurações > Contatos:** É expressamente proibido acessar a área de contatos ou transferências no WidePay. Usar apenas as áreas financeiras (Carnês e Cobranças/Boletos).
4. **Não Confundir Iniciais com Quadras:** As letras "A a E" solicitadas nas etapas representam as iniciais dos nomes dos clientes (Ex: Ana, Belmiro, Camila, etc.), não as quadras físicas A, B, C, D e E.
5. **Login com Autopreenchimento Seguro:** Se o WidePay abrir na tela de login e as credenciais estiverem salvas e visíveis no navegador, o Antigravity pode simular um clique de hardware (coordenadas) no campo de senha para ativá-lo e clicar em "Acessar". Não ler, salvar ou copiar senhas.
6. **Não Criar Timers Repetidos:** Em caso de espera por processamento, reportar o status real uma única vez e aguardar decisão do usuário. Sem timers em loops.
7. **Não Gerar Arquivos Finais sem Aprovação:** Apresentar a conferência em Markdown no chat antes de gerar PDFs ou HTMLs de entrega.
8. **Não Alterar Nada no WidePay:** O acesso ao WidePay é estritamente de somente leitura (read-only). Nunca crie, cancele ou edite cobranças.
9. **Não Usar Contratos como Lista Principal:** A lista de clientes avaliados deve ser extraída das informações financeiras do WidePay, cruzando depois com os contratos locais.
10. **Não Inventar Valores:** Registre apenas dados comprovados no WidePay ou contratos físicos. Dúvidas devem ser reportadas como "pendente de conferência".
11. **Somar Todos os Carnês e Boletos:** Antes de informar o total pago geral de um cliente, localize e some todas as cobranças e carnês ativos vinculados a ele, não apenas o primeiro/atual.
12. **Registrar Arquivos Locais x GitHub:** Manter controle dos commits e links reais para todos os arquivos enviados para o repositório remoto.
